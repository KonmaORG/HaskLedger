# Governance

HaskLedger is developed and maintained by [KonmaORG](https://github.com/KonmaORG).

## Maintainer

**Vinit Inamke** (@VINIT-INAMKE) - project lead

## How decisions are made

Day-to-day decisions (bug fixes, docs, issue triage, releases) are handled by
the maintainer. For anything significant - architectural changes, breaking API
changes, new features, guideline changes - community input is welcome via
GitHub Issues or Discussions.

## Communication

- [GitHub Issues](https://github.com/KonmaORG/HaskLedger/issues) - bugs and feature requests
- [GitHub Discussions](https://github.com/KonmaORG/HaskLedger/discussions) - general discussion
- Pull requests - code contributions

## Response times

- Security issues: 48 hours
- Bugs: 3–5 days
- PRs and general issues: 1–2 weeks

## Releases

Features are planned, implemented, and tested against all four example
contracts before tagging a release.

## Code of conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md).
